# NearbyPulse Pro 附近脉动 - Telegram Mini App 需求文档

## 1. 项目背景与目标
**项目名称**：NearbyPulse Pro（附近脉动）
**项目定位**：基于 Telegram 生态构建的“本地商户发现与众包平台”。
**核心目标**：
1. **解决信息差**：帮助 Telegram 用户快速发现周边经过验证的高质量实体门店（如餐饮、咖啡馆、服务业等）。
2. **极低获客成本**：借助 Telegram 社交生态的免密登录特性，让本地商户零门槛接驳线上，快速对接庞大流量。
3. **众包共建引擎**：引入积分激励机制，鼓励海量普通用户作为“数字探店员”，主动发掘、上传当地优质店面，从而实现产品的高速自增长。

---

## 2. 目标用户态与使用场景
- **普通用户 (C端顾客)**：周末不知道去哪儿玩，打开该小程序，浏览附近最新高分店铺，标记为“待打卡”。真实到店后，通过内置 Telegram 一键联系老板定位并发评价。
- **探店贡献者 (C端推广员)**：路过一家新开的网红奶茶店，随手用小程序拍照、地图选点收录，获取平台积分奖励。通过积分机制提升参与度。
- **商户老板 (B端商家)**：自己的店铺被上传或主动入驻后，可以直接把店铺的专属详情页丢在周边社区的 Telegram 群组里。由于是内嵌小程序，群友点击即刻展示全貌并可直接私信咨询下单。老板自己还能随时通过身份校验（Tg ID）进行店面位置微调和文字编辑。

## 3. 核心功能模块 (Core Modules)

### 3.1 “发现”瀑布流模块 (Explore Feed)
此模块为进入小程序的默认首屏。
- **动态发现列表**：按照最新倒序拉取呈现，展示所有审核通过的高质量本地商户卡片。
- **高阶视觉与智能补缺**：每张商户卡都标配超大横向图面。拥有“智能兜底图机制” —— 即便店面没有真实照片，系统也会根据其选择的“经营类目”（如“咖啡茶饮”、“美容健身”）自动加载高质感的缺省图封，确保整个页面的 UI 品质极佳。
- **快捷互动区**：用户可点击卡片表面的 `[待打卡]` 和 `[已打卡]`，快速归置到个人的数字备忘录。
- **宏观数据总览**：动态展示当前平台的总入驻商户数、当周推荐频率等运营看板。

### 3.2 商家详情与评论互动系统 (Detailed Profile & Reviews)
在“发现”流中点击图片进入平滑弹出的专属详情页。
- **基本面概览**：展示商户实景封面图、店名、认证状态、专属分类及地址。
- **地图缩略展示（静态辅助）**：纯净呈现场景中，提供一张 21:9 极具高级感的 Leaflet 静态渲染地图，快速建立消费者的地理位置第一印象。
- **社交直连通道**：若系统中有配置老板的 Telegram 账号（`tg_user_id`），直接呈现极其显眼的 `联系老板` 功能，一键直达 Telegram 聊天窗口。
- **防篡改的店家自治 (Owner Editor)**：基于无感知的会话验证体系（通过比对 `submitter_tg_id`），详情页会自动判断来访者是否为创建人。是的话则释放隐秘的【编辑模式】，支持对店铺精确经纬度和特色文字进行即时更新。不是创建人的游客无法查阅任何编辑入口。
- **共创评价系统 (Rewarding Reviews)**：
  - 允许所有通过 Telegram 访问的去中心化账户撰写留言和看店建议。
  - **红心点赞（Like）**：社交强化，一键点赞优质评价且自动防重复。
  - **互动货币化（Rewards）**：发表评价被认定为正向促活操作，随即便可获取随机（目前为代码层生成）积分，系统同时伴随沉浸式的动画和礼花弹窗。

### 3.3 “推荐”收录模块 (Crowdsource Form)
采用分步渐进式 (4 Steps Progress) 向导的标准化提交界面。
- **Step 1 (基本信息设定)**：登记商铺名称、类型下拉选项及特色介绍大纲。
- **Step 2 (电报关联通道)**：输入或粘贴商铺运营者的 @Handle。
- **Step 3 (动态传图)**：对接高并发云端架构 (Supabase Storage Bucket)，实现本地相册原生级多图、甚至是短视频（待定）调取上传与鉴权。
- **Step 4 (空间选址与搜索)**：
  - 加载基于开源技术的世界坐标图盘 (Leaflet Interactive Map)。
  - **聚合检索引擎 (Nominatim)**：支持手打地名（市、区、具体店名），系统执行跨国界的模糊搜索并动态驱使相机构图点平滑飞行穿梭（FlyTo）至对应的精准街道上。支持人工再拖拽确认。
- **推荐闭环 (Earn Base)**：完成向导后触发系统奖励池动作，为当前用户直增 20 个人积分。

### 3.4 “我的”个人数字空间 (User Control Center)
完全属于用户个体的数字档案集合站。
- **Telegram Native Profile**：零密码拦截下，全自动拉取并绘制账户头像、昵称，在抬头处悬挂最重要的资产维度：“现有总积分”。
- **资产明细日志 (Points Ledgers)**：利用底层金融思维搭建日志记录级历史长轴。展示所有收支流水情况与动作源头（例如：“+20分：推荐新店-星巴克”，“+7分：发布评价-沙县”）。包含精确的时间刻印及账套分离颜色处理（绿营增、红营减）。
- **收藏双栈分流 (Bookmarks Pool)**：
  - 横向 Tab 控制“待打卡”、“已打卡”两套账本。
  - 采用纵向列表重排列逻辑布局，卡片依旧继承强大的全局后补图计算规则，支持直接在卡面移除整理或唤醒老板私窗。

## 4. UI与交互设计规范 (UI/UX Protocol)
- **新拟态流体系统 (Glassmorphism Next)**：大面积抛弃死板块区划，依靠 Tailwind 定义的半透明复合堆积态背景（`bg-white/90`、`backdrop-blur-xl`），极简主义细线边框配合底影（Shadows），呈现 iOS 及 Next-Gen Web 生效质感。
- **情感反馈与缓冲动效**：界面核心渲染管线引入 `Framer-motion` 全局管理体系。路由与内容切换全无刷新感，取而代之的是阻尼弹入、平滑淡出、缩放补间；各种 Tab 底线具备“活态追踪”特性。
- **完全原生的 Telegram 感觉接管**：充分调动 `@twa-dev/sdk` 能力接管底层原生宿主特性：包括原生主控撤退级“物理按键（BackButton）”，全套触觉力震动回馈引擎 (Haptic Feedback)（如：红心点亮引发的高雅“Light”级轻震，或是数据验证后的重“Success”级震感），以及深层嵌套原生风格弹体（Confirm / ShowPopup）维持统一的心流氛围。

## 5. 项目技术群与架构搭建选型
- **客户端基石**：React 18 + Vite 高速编译器组合。单页运用 SPA 打法。
- **重器组件库构筑**：基于定制化的 Tailwind CSS v3 并集成跨系统字体族系方案；使用开源矢量化顶级字偶距图标库 Lucide-React。
- **动画中间件**：Framer-motion。
- **电报联结件**：`@twa-dev/sdk`。作为获取用户鉴权 Token 的首要安全关卡。
- **动态寻星系统（地图）**：放弃了复杂昂贵的谷歌地图鉴戒。转向采用 Leaflet + React-Leaflet 4.x 并混入了 OpenStreetMap (OSM) 开源图库与检索服务 API，达成纯前端零成本的地理编码。
- **全托管云后方工程库 (Supabase BaaS System)**：
  - **核心数据库**：通过云端托管处理的高性能 PostgreSQL 引擎。
  - **储存大库区**：依靠 Storage 构件做富媒体处理与 CND 放投。
  - **表层物理安全锁定 (RLS)**：严格依赖行级安全策略管控所有的数据库插入更新权限，防御基础越权攻击。

## 6. 顶层数据库表述 (Database Schema Layout)

针对项目核心运转诉求，设立四大实体联结。

1. **`merchants` (商机母表)**:
   - 全盘商机运转枢轴：[id](file:///Users/wangzenghui/Documents/TelegramApp/src/components/AddMerchantForm.jsx#133-137) (主键 UUID)、`name`、`category`、`physical_address`、`description` (详情补充)、`geo` (二维坐标组)、`owner_tg_id` (商家绑定的 Telegram Account)、`submitter_tg_id` (创生者鉴定码，用作隔离编削特权)、`media_urls` (JSONB 图片数据池)、`status` (准入校验池标志)。
2. **`reviews` (点评评价栈)**:
   - 共创核心内容库：[id](file:///Users/wangzenghui/Documents/TelegramApp/src/components/AddMerchantForm.jsx#133-137)、`merchant_id` (联结目标店面)、`tg_user_id` (点评人)、`tg_username`、`content`、`likes` 统计池、`created_at`。
3. **`review_likes` (防刷量多维表)**:
   - 行使双位关联查重：主键锁死为 [(review_id, tg_user_id)](file:///Users/wangzenghui/Documents/TelegramApp/src/App.jsx#18-157)，遏制并发刷赞。
4. **`points_history` (积分账本体)**:
   - 行使经济行为回溯的源表：追踪项包括 `tg_user_id`、动作源 `action`、产生币值 `points` 以及时间截面 `created_at`。
